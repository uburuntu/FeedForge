# FeedForge v0.3.0

FeedForge v0.3.0 establishes the compatibility and resource contracts that the
1.0 line will build on. It remains experimental, is not exchange-certified,
and is not production trading infrastructure.

## Highlights

- Runtime/generated compatibility now uses an explicit API epoch and minimum
  revision instead of one exact integer.
- Replay counters and absolute error offsets use `std::uint64_t`; chunked
  framing reports terminal `offset_overflow` rather than wrapping.
- `feedforgec` has one-output `validate`, `compile`, and `dump-ir` commands,
  stable exit statuses, and a guarded internal-error path.
- Compiler inputs, TOML structure, model cardinality, identifiers,
  documentation, and rendered output have published deterministic limits.
- Canonical headers and FFIR fixtures were regenerated by the v0.3 compiler.

## Compatibility

The package and generated-header identity are `0.3.0`. Runtime API epoch 1,
revision 0 is a clean source boundary: v0.2's `runtime_api_version` member is
not retained. Regenerate custom headers and rebuild consumers when upgrading.
Installed CMake packages continue to use same-minor compatibility before 1.0.

Schema, pipeline, and FFIR format versions remain 1, and
`portable_checked.v1` remains the only implementation profile. Canonical FFIR
JSON remains deterministic diagnostic output, not a long-term public
interchange guarantee.

Runtime and generated headers remain strict C++20. The optional host compiler
remains C++23. See [Compatibility](docs/compatibility.md),
[Compiler CLI](docs/compiler-cli.md), and
[Compiler limits](docs/compiler-limits.md) for the durable contracts.

## Limitations

- FeedForge provides no live networking, packet recovery, sequencing, order
  book, strategy, capture service, database, or operational trading controls.
- Input is caller-owned in-memory data or caller-delivered chunks. FeedForge
  performs no file or network I/O.
- Exchange data is not bundled. Authoritative protocol documents remain
  subject to their owners' terms.

See the [v0.2.0 GitHub Release](https://github.com/uburuntu/FeedForge/releases/tag/v0.2.0)
for the previous release notes and validation evidence.
