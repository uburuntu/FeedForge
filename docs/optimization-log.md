## Iteration 01 — force-inline generated decode entry point

Status: accepted.

- Hypothesis: AppleClang leaves the large sink-specialized `decode_one` out of
  line (confirmed by a local symbol in the baseline executable). A guarded
  force-inline annotation should remove one hot call/return per message and
  expose payload/sink state to the caller without changing validation order or
  outcomes.
- Exact source files: `include/feedforge/config.hpp`,
  `src/feedforgec/emit_cpp.cpp`.
- Frozen target: `decode_one/itch50_order_events/mixed`.
- Expected mechanism: compiler-supported `always_inline`/`__forceinline` with a
  portable plain-`inline` fallback; canonical headers will be regenerated by
  the generator.

## Iteration 06 — exact-width runtime byte assembly

Status: rejected and reverted.

- Hypothesis: compiler remarks confirm all `load_unsigned` instantiations inline
  in the timed decode path, while assembly shows the ubiquitous six-byte ITCH
  timestamp load expanding to six byte loads and a serial shift/OR chain.
  A constant-evaluation-preserving runtime path based on exact-width
  `memcpy` plus portable endian normalization should let the compiler combine
  the six-byte read into wider legal byte copies without reading past the
  validated field.
- Exact source file: `include/feedforge/wire/load_big_endian.hpp`.
- Frozen target: `decode_one/itch50_all/all_types`.
- Expected mechanism: retain the current byte loop during constant evaluation;
  at runtime copy exactly `Width` bytes into a zero-initialized `uint64_t`,
  normalize native little/big endian with a recognized portable byte swap, and
  shift the result into place. This performs no typed unaligned dereference and
  reads no byte outside the already size-validated field.

Comparator verdict: `optimization_win=false`.

### Exact retained changes

1. `include/feedforge/config.hpp` defines a compiler-guarded
   `FEEDFORGE_FORCE_INLINE`; generated `decode_one` uses it so the sink-specific
   decoder is inlined without changing the portable fallback.
2. `src/feedforgec/emit_cpp.cpp` emits a single size-validation/event-dispatch
   switch when a pipeline selects every schema message.
3. For partial pipelines, the emitter dispatches selected messages first and
   sends only misses through the known-unselected size/skip switch.
4. For partial pipelines only, generated replay uses a direct, bounds-checked
   framing loop; all-selected replay retains `binary_file_cursor`.

There are no retained packed layouts, unaligned typed dereferences,
architecture-specific flags, allocation, runtime polymorphism, read-before-
size-check behavior, or benchmark-only shortcuts. Iterations 06, 07, and 10
left no retained byte-load or direct-aggregate changes.
