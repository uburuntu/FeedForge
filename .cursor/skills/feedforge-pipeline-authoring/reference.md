# FeedForge pipeline reference

## Format-version-1 grammar

Unknown keys are errors at every level.

### Required top-level keys

| Key | Accepted value |
|---|---|
| `format_version` | Integer `1` |
| `name` | `[a-z][a-z0-9_]*`, C++-safe |
| `namespace` | One or more valid C++ namespace components separated by `::` |
| `schema` | Exact `name` from the parsed schema |
| `profile` | `"portable_checked"` |
| `unknown_messages` | `"error"` or `"skip"` |
| `unselected_messages` | `"skip"` |

At least one `[[emit]]` is required.

### Required emit keys

| Key | Accepted value |
|---|---|
| `source` | Exact one-byte discriminator from the schema |
| `event` | Unique `[a-z][a-z0-9_]*`, C++-safe |
| `fields` | Nonempty unique explicit field names, or exactly `["*"]` |

One source discriminator may be selected only once. Event names must be unique and must not collide with generated declarations. C++ keywords, leading underscores, names containing `__`, and reserved generated names are rejected.

Discriminator and `reserved` fields are not projectable. Wildcard expansion selects every projectable field in wire-offset order. Explicit selection keeps the listed order even when it differs from wire order.

## Selection decisions

Prefer explicit fields when:

- the consumer has a narrow data contract;
- generated layout is reviewed field by field;
- adding a schema field must not silently expand the event; or
- field/member order is intentionally controlled.

Use `["*"]` when the event intentionally mirrors all projectable schema fields and future reviewed schema additions should enter the event.

Field order affects aggregate member order, object layout, FFIR, the pipeline semantic fingerprint, and generated bytes. Do not sort an explicit projection unless changing that contract is intentional.

Emit-table order does not affect output; lowering sorts emits by unsigned discriminator. TOML key order and comments are also non-semantic.

## Runtime policy consequences

Known messages are always checked against their exact schema size.

- Selected + valid: construct and emit the projected event.
- Known unselected + valid: `known_unselected_skipped`.
- Unknown + `"error"`: `unknown_message_type`.
- Unknown + `"skip"`: `unknown_skipped`.
- Any known wrong-sized message: `invalid_message_size`, regardless of policy.

Use `"error"` unless the application explicitly accepts discriminators absent from the schema. `unselected_messages` has no alternative to `"skip"` in format version 1.

## Preserved C++ value types

Projection does not change the schema's logical type:

| Schema logical type | Generated C++ type |
|---|---|
| `raw_unsigned`, width 1/2/4/6/8 | `uint8_t`/`uint16_t`/`uint32_t`/`uint64_t`/`uint64_t` |
| `ascii` or `alpha` | `feedforge::ascii<N>` |
| `timestamp_ns` | `feedforge::timestamp_ns` |
| `decimal`, width 4/8, scale `S` | `feedforge::decimal<uint32_t, S>` / `feedforge::decimal<uint64_t, S>` |
| `stock_locate` | `feedforge::stock_locate` |
| `tracking_number` | `feedforge::tracking_number` |
| `order_reference_number` | `feedforge::order_reference_number` |
| `match_number` | `feedforge::match_number` |
| `share_count` | `feedforge::share_count` |

There is no pipeline syntax for casts, aliases, field renames, optional values, variants, strings, vectors, expressions, or computed fields.

## Compiler CLI

Build a compiler-enabled source tree:

```sh
cmake -S /path/to/FeedForge -B build/feedforge-tools \
  -DFEEDFORGE_BUILD_COMPILER=ON \
  -DFEEDFORGE_BUILD_TESTS=OFF \
  -DFEEDFORGE_BUILD_EXAMPLES=OFF
cmake --build build/feedforge-tools --target feedforgec
```

Schema-only validation:

```sh
build/feedforge-tools/src/feedforgec/feedforgec validate \
  --schema /path/to/schema.toml
```

Schema-bound pipeline validation:

```sh
build/feedforge-tools/src/feedforgec/feedforgec validate \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml
```

Generation:

```sh
cmake -E make_directory build/generated
build/feedforge-tools/src/feedforgec/feedforgec compile \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/generated/pipeline.hpp

build/feedforge-tools/src/feedforgec/feedforgec dump-ir \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/generated/pipeline.ffir.json
```

The compiler does not create the CLI output's parent directory. Each invocation writes one output atomically and avoids changing an existing output when validation fails. `--output` must not name the schema or pipeline.

Successful `validate`, `compile`, and `dump-ir` commands produce no standard output or error text. `--help` and `--version` are standalone commands.

## Diagnostics and exit status

The stable rendering is:

```text
FFPIPE018 path/to/pipeline.toml:12:5: pipeline.emit.event.fields: message '...' has no field named '...'
hint: optional remediation
```

The position or hint can be absent. Paths use `/` separators; object paths identify the model element even when a text position is unavailable.

| Prefix/code | Layer |
|---|---|
| `FFTOML001` | TOML syntax/parser failure |
| `FFSCHEMA...` | Schema grammar, identifier, type, or layout validation |
| `FFPIPE...` | Pipeline grammar or schema-bound semantic validation |
| `FFCLI...` | Command/option or path-alias misuse |
| `FFIO001` | Unreadable input |
| `FFIO002` | Output parent/write/replace failure |
| `FFLIMIT001` | Published compiler resource limit exceeded |
| `FFEMIT001` | C++ emission invariant failure |
| `FFINTERNAL001` | Unexpected internal compiler failure |

Common pipeline codes:

| Code | Meaning |
|---|---|
| `FFPIPE001` | Missing/unsupported format version |
| `FFPIPE002` | Unknown key |
| `FFPIPE003` | Missing required key |
| `FFPIPE004` | Wrong TOML value type |
| `FFPIPE005` | Invalid pipeline/event/schema identifier |
| `FFPIPE006` | Invalid/reserved namespace |
| `FFPIPE007` | Pipeline/schema name mismatch |
| `FFPIPE008` | Unsupported profile |
| `FFPIPE009` | Unsupported policy |
| `FFPIPE010` | No emit tables |
| `FFPIPE011` | Source discriminator is not one byte |
| `FFPIPE012` | Unknown source discriminator |
| `FFPIPE013` / `FFPIPE014` | Duplicate source / event |
| `FFPIPE015` / `FFPIPE016` | Empty projection / invalid wildcard use |
| `FFPIPE017` / `FFPIPE018` | Duplicate / unknown projected field |
| `FFPIPE019` | Discriminator or reserved field projected |
| `FFPIPE020` | Field lacks a C++20 representation |
| `FFPIPE021` | Collision with a generated declaration |

Exit status is `0` for success, `2` for invalid CLI/input/emission, `3` for input/output I/O failure, and `4` for an unexpected internal failure. Automation should key on exit status and stable code, while still showing the full location, object path, message, and hint to a human.

## CMake generation contract

After `find_package(FeedForge CONFIG REQUIRED)` or compiler-enabled FetchContent:

```cmake
if(NOT TARGET FeedForge::compiler)
  message(FATAL_ERROR "Custom FeedForge generation needs the host compiler")
endif()

feedforge_generate(
  NAME market_events
  SCHEMA nasdaq_totalview_itch_5_0
  PIPELINE "${CMAKE_CURRENT_SOURCE_DIR}/market_events.toml"
)

target_link_libraries(
  consumer
  PRIVATE FeedForge::generated::market_events
)
```

The symbolic schema name resolves through `FeedForge_SCHEMA_DIR`. An absolute schema path and a source-directory-relative schema path are also accepted.

The default output is generated under the build tree and included as:

```cpp
#include <feedforge/generated/market_events.hpp>
```

The pipeline's TOML namespace controls C++ names. `NAME` controls the CMake target and default header filename.

## Determinism check

```sh
cmake -E make_directory build/determinism/a build/determinism/b

build/feedforge-tools/src/feedforgec/feedforgec compile \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/determinism/a/pipeline.hpp

build/feedforge-tools/src/feedforgec/feedforgec compile \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/determinism/b/pipeline.hpp

build/feedforge-tools/src/feedforgec/feedforgec dump-ir \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/determinism/a/pipeline.ffir.json

build/feedforge-tools/src/feedforgec/feedforgec dump-ir \
  --schema /path/to/schema.toml \
  --pipeline /path/to/pipeline.toml \
  --output build/determinism/b/pipeline.ffir.json

cmake -E compare_files \
  build/determinism/a/pipeline.hpp \
  build/determinism/b/pipeline.hpp
cmake -E compare_files \
  build/determinism/a/pipeline.ffir.json \
  build/determinism/b/pipeline.ffir.json
```

For a semantic-reordering check, repeat with copies that differ only in comments, key order, and emit-table order. Do not reorder explicit `fields`; that changes semantics.

Generated output must not contain absolute input paths, generation time, hostname, username, or random identifiers. Metadata should contain the generator version, FFIR version, schema/pipeline fingerprints, and profile variant.

## Custom CMake output

`feedforge_generate(OUTPUT ...)` resolves a relative path from the current
binary directory and requires a literal path inside the top-level build tree.
The filename must be exactly `NAME.hpp`.

An output ending in `feedforge/generated/NAME.hpp` exports the directory before
`feedforge`, so `<feedforge/generated/NAME.hpp>` remains valid. Other matching
locations preserve the legacy flat `<NAME.hpp>` include from the immediate
parent. The helper rejects mismatched names, input aliases, and duplicate or
unsafe outputs during configure.
